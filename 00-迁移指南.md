## 📋 迁移步骤

### Step 1：安装 OpenClaw（本地电脑）
```bash
# 确保 Node.js v20+ 已安装
npm install -g openclaw
openclaw init
```

### Step 2：复制核心文件
将以下文件从 `migration/` 复制到本地 workspace 根目录（通常是 `~/.openclaw/workspace/`）：

| 源文件 | 目标位置 | 说明 |
|--------|----------|------|
| `01-AGENTS.md` | `workspace/AGENTS.md` | 工作区主配置 |
| `02-SOUL.md` | `workspace/SOUL.md` | 婉儿灵魂 |
| `03-IDENTITY.md` | `workspace/IDENTITY.md` | 婉儿身份 |
| `04-USER.md` | `workspace/USER.md` | 用户信息 |
| `05-TOOLS.md` | `workspace/TOOLS.md` | 工具配置（**需修改**） |
| `06-PRINCIPLES.md` | `workspace/PRINCIPLES.md` | 工作原则 |
| `07-HEARTBEAT.md` | `workspace/HEARTBEAT.md` | 心跳检查 |
| `08-MEMORY.md` | `workspace/MEMORY.md` | 长期记忆 |

### Step 3：复制三省六部目录
将整个 `migration/三省六部/` 复制到 `workspace/三省六部/`

### Step 4：修改本地配置（TOOLS.md）
本地电脑的 TOOLS.md 需要根据实际情况修改：
- **搜索 API**：本地可用 SearXNG 或 Brave Search
- **消息通道**：根据本地需求配置（钉钉/飞书/Telegram 等）
- **TTS**：Edge TTS 免费可用
- **时区**：如不在中国，需修改时区
- **位置**：修改为你的城市

### Step 5：配置 OpenClaw 模型
在 OpenClaw config 中配置模型（与服务器一致）：
```yaml
model: dashscope/qwen3.6-plus
# 或其他可用模型
```

### Step 6：启动验证
```bash
openclaw start
# 验证婉儿是否正常加载
# 说"婉儿，早上好"测试
```

## ⚠️ 注意事项

1. **不包含敏感信息** — 迁移包中不包含 API Key、Token、密码等
2. **需重新配置 OAuth** — Google Calendar、Gmail 等 OAuth 需要重新授权
3. **memory/ 目录未包含** — 日常日志文件太大，建议按需复制最近几天的
4. **钉钉/飞书等凭证** — 需要在本地重新配置
5. **子代理配置** — 三省六部的 subagent 需要在本地重新创建
